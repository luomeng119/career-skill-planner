# 内容运营 Skill 规划输出示例

> 以下是在 Claude Code 中说"我是内容运营，帮我拆 Skill"后，career-skill-planner 输出的完整清单。

---

⚠️ 以下基于「内容运营」的常见职责拆解。如果实际工作有差异，告诉我，我来调整。

---

1) 内容策划 Skill（co-content-planning）

- 用户提供什么
  - 内容领域和定位、目标受众、发布渠道、热点事件或话题方向

- 提示词（可直接用）
  - 「创建 co-content-planning Skill。制定内容策略和选题规划，结合热点和用户需求确定内容方向和排期。输出：内容定位和策略、选题清单（含热度评估）、内容矩阵（形式/频率/渠道）、月度/周度排期表、热点响应预案。防呆要求：如果热点信息不足，提供选题挖掘框架，引导用户基于自身领域补充。」

- 实现逻辑
  - 趋势分析 → 选题挖掘 → 内容矩阵规划 → 排期制定 → 效果预测。

---

2) 文案撰写 Skill（co-copywriting）

- 用户提供什么
  - 文案类型和场景、品牌调性和关键词、目标受众、字数或格式要求、参考文案（可选）

- 提示词（可直接用）
  - 「创建 co-copywriting Skill。根据品牌调性和传播目标撰写高质量的文案，包括标题、正文、Slogan 等。输出：5-10 个备选标题 + 推荐理由、完整文案 + 关键词强调、5-10 个备选 Slogan + 适用场景。防呆要求：如果品牌调性不明确，基于行业类型推荐调性方向，提供选项让用户确认。」

- 实现逻辑
  - 需求理解 → 调性分析 → 创意发散 → 文案生成 → 优化迭代。

---

3) 数据报告 Skill（co-data-report）

- 用户提供什么
  - 数据源和指标、报告类型（日报/周报/月报/专题）、目标受众、时间范围

- 提示词（可直接用）
  - 「创建 co-data-report Skill。将运营数据转化为有洞察的报告，包括周报/月报、专题分析和数据可视化方案。输出：核心指标概览、趋势变化分析、异常数据说明、关键洞察（3-5 条）、行动建议、附录（数据表格）。防呆要求：如果数据不完整，基于可获取的数据进行分析，标注[数据缺失]，不影响主流程。」

- 实现逻辑
  - 数据收集 → 趋势分析 → 异常识别 → 洞察提炼 → 报告撰写。

---

4) 渠道运营 Skill（co-channel-ops）

- 用户提供什么
  - 运营渠道列表、内容素材、各平台规则和受众特点、运营目标（阅读/互动/转化）

- 提示词（可直接用）
  - 「创建 co-channel-ops Skill。管理多渠道内容分发和运营，包括公众号、抖音、小红书、微博等。输出：渠道矩阵和定位、各平台内容适配规则、发布排期表、互动策略、数据监控指标、优化迭代计划。防呆要求：如果平台规则不明确，提供主流平台（微信/抖音/小红书）的基础规则作为参考。」

- 实现逻辑
  - 渠道盘点 → 内容适配 → 发布排期 → 数据监控 → 优化迭代。

---

5) SEO 优化 Skill（co-seo-optimizer）

- 用户提供什么
  - 目标 URL 或内容、目标关键词、竞品参考（可选）、内容类型

- 提示词（可直接用）
  - 「创建 co-seo-optimizer Skill。优化内容在搜索引擎中的排名，包括关键词研究、内容优化和技术 SEO 建议。输出：关键词分析（搜索量/竞争度/相关性）、内容优化建议（标题/正文/Meta）、内部链接建议、技术 SEO 检查、排名提升预测。防呆要求：如果目标关键词不明确，基于内容主题推荐关键词方向。」

- 实现逻辑
  - 关键词分析 → 内容审计 → 优化建议 → 技术检查 → 效果预测。

---

6) 内容日历 Skill（co-content-calendar）

- 用户提供什么
  - 发布渠道、内容类型和频率、团队产能、特殊时间节点（节日/活动）

- 提示词（可直接用）
  - 「创建 co-content-calendar Skill。制定和管理内容发布日历，确保内容产出的持续性和节奏感。输出：发布日历表（日/周/月视图）、内容类型分布、负责人分配、特殊节点标注、缓冲时间设置。防呆要求：如果团队产能不明确，基于常见内容团队（1人/3人/5人）给出产能参考。」

- 实现逻辑
  - 产能评估 → 内容主题分配 → 排期制定 → 日历生成 → 提醒机制。

---

以上 6 个 Skill 覆盖了内容运营的核心工作流。

使用方式：把上面任意一条"提示词"直接发给 Agent（比如 Claude Code），它会帮你生成对应的 Skill 文件。

建议优先从最耗时、最高频的场景开始，做完一个用起来，再做下一个。

如果这份清单里有不符合实际的场景，告诉我，我来替换。
