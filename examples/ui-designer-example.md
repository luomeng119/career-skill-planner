# UI 设计师 Skill 规划输出示例

> 以下是在 Claude Code 中说"我是 UI 设计师，帮我拆 Skill"后，career-skill-planner 输出的完整清单。

---

⚠️ 以下基于「UI 设计师」的常见职责拆解。如果实际工作有差异，告诉我，我来调整。

---

1) 设计系统 Skill（ds-design-system）

- 用户提供什么
  - 现有设计素材和组件、品牌色值和字体规范、目标平台（Web/iOS/Android）、团队规模和协作方式

- 提示词（可直接用）
  - 「创建 ds-design-system Skill。建立和维护一套完整的设计系统，包括设计原则、组件库、样式规范和文档。输出：设计原则（3-5 条）、颜色系统（Token 定义）、字体系统、间距系统、组件清单、使用指南、维护和迭代流程。防呆要求：如果品牌规范不完整，基于常见设计模式给出建议方案并标注[假设]。」

- 实现逻辑
  - 设计原则定义 → Token 体系建立 → 组件库规划 → 文档编写 → 维护机制。

---

2) 线框图 Skill（ds-wireframe）

- 用户提供什么
  - 页面目标和使用场景、需要展示的内容和功能、参考页面（可选）、技术约束

- 提示词（可直接用）
  - 「创建 ds-wireframe Skill。快速生成页面线框图，包括布局结构、信息层级和交互逻辑。输出：页面结构说明、区域划分、信息层级和元素布局、交互说明。防呆要求：如果需求描述不完整，基于常见页面模式给出建议结构，标注[假设]。」

- 实现逻辑
  - 需求理解 → 信息架构 → 布局设计 → 交互标注 → 迭代反馈。

---

3) 视觉评审 Skill（ds-visual-review）

- 用户提供什么
  - 设计稿（Figma 链接 / 截图 / 描述）、设计系统规范、评审维度（可选）

- 提示词（可直接用）
  - 「创建 ds-visual-review Skill。对设计稿进行系统性视觉评审，检查一致性、可用性和美观度。输出：整体评价、一致性检查（间距/颜色/字体）、可用性问题、视觉美观度评价、逐条改进建议（按优先级 P0/P1/P2）。防呆要求：如果未提供设计系统规范，基于常见设计原则进行评审，标注[基于通用设计原则]。」

- 实现逻辑
  - 视觉检查 → 一致性验证 → 可用性评估 → 美观度判断 → 改进建议。

---

4) 组件库 Skill（ds-component-library）

- 用户提供什么
  - 组件需求和场景、设计系统规范、技术栈（React/Vue/SwiftUI 等）、可访问性要求

- 提示词（可直接用）
  - 「创建 ds-component-library Skill。设计和管理 UI 组件库，确保组件的可复用性、一致性和可访问性。输出：组件名称和用途、组件变体（variants）、状态（default/hover/active/disabled/loading/error）、尺寸和响应式行为、可访问性要求、使用示例、Do's and Don'ts。防呆要求：如果技术栈信息缺失，提供主流框架的实现建议并标注[假设]。」

- 实现逻辑
  - 需求分析 → 组件设计 → 状态定义 → 文档编写 → 示例代码。

---

5) 交互设计 Skill（ds-interaction-design）

- 用户提供什么
  - 交互场景和用户流程、功能需求和状态变化、性能约束、品牌调性

- 提示词（可直接用）
  - 「创建 ds-interaction-design Skill。设计流畅的用户交互流程，包括微交互、过渡动画和状态转换。输出：用户流程图、交互节点和触发条件、动效参数（时长/缓动/层级）、状态转换矩阵、错误和边界状态处理。防呆要求：如果品牌调性不明确，基于功能类型推荐适合的动效风格，标注[假设]。」

- 实现逻辑
  - 用户流程梳理 → 交互节点识别 → 动效设计 → 状态转换定义 → 原型制作。

---

6) 设计交付 Skill（ds-design-handoff）

- 用户提供什么
  - 设计稿（Figma / Sketch / 截图）、目标平台和技术栈、设计系统规范、特殊交互说明

- 提示词（可直接用）
  - 「创建 ds-design-handoff Skill。将设计稿转化为研发可用的交付物，包括标注、切图、交互说明和开发指南。输出：设计说明文档、标注清单（间距/字体/颜色/圆角）、切图清单（格式/倍率/命名）、交互说明、状态清单、边界情况和异常状态、开发自检清单。防呆要求：如果技术栈信息缺失，提供多平台切图建议（Web 1x/2x、iOS @2x/@3x、Android mdpi/hdpi/xhdpi）。」

- 实现逻辑
  - 设计审核 → 标注生成 → 切图导出 → 交互文档 → 交付检查。

---

以上 6 个 Skill 覆盖了 UI 设计师的核心工作流。

使用方式：把上面任意一条"提示词"直接发给 Agent（比如 Claude Code），它会帮你生成对应的 Skill 文件。

建议优先从最耗时、最高频的场景开始，做完一个用起来，再做下一个。

如果这份清单里有不符合实际的场景，告诉我，我来替换。
