# 前端工程师 Skill 规划输出示例

> 以下是在 Claude Code 中说"我是前端工程师，帮我拆 Skill"后，career-skill-planner 输出的完整清单。

---

⚠️ 以下基于「前端工程师」的常见职责拆解。如果实际工作有差异，告诉我，我来调整。

---

1) 代码审查 Skill（fe-code-review）

- 用户提供什么
  - 代码文件或 PR 链接、技术栈（React/Vue/Angular/Svelte）、审查重点（性能/安全/架构/规范）

- 提示词（可直接用）
  - 「创建 fe-code-review Skill。对前端代码进行系统性审查，检查代码质量、性能、安全性和最佳实践。输出：整体评价（通过/有条件通过/不通过）、架构问题、安全问题、性能问题、可维护性问题、逐行改进建议、正面反馈。防呆要求：如果代码量过大，分模块审查，优先审查核心逻辑和边界情况。」

- 实现逻辑
  - 代码阅读 → 架构检查 → 安全审查 → 性能分析 → 可维护性评估 → 改进建议。

---

2) 组件设计 Skill（fe-component-design）

- 用户提供什么
  - 组件需求和场景、设计稿或交互说明、技术栈和框架、可访问性要求

- 提示词（可直接用）
  - 「创建 fe-component-design Skill。设计高质量的前端组件，包括 API 设计、状态管理和可访问性。输出：组件 API（Props 定义）、组件状态管理、事件处理、样式方案、可访问性要求、使用示例、边界情况处理。防呆要求：如果需求不明确，基于常见组件模式给出建议方案，标注[假设]。」

- 实现逻辑
  - 需求分析 → API 设计 → 状态设计 → 可访问性 → 文档编写 → 示例代码。

---

3) 性能优化 Skill（fe-performance-optimize）

- 用户提供什么
  - 性能问题描述、Lighthouse 报告或 Web Vitals 数据、目标指标、技术栈

- 提示词（可直接用）
  - 「创建 fe-performance-optimize Skill。诊断和优化前端性能，包括加载性能、运行时性能和 Core Web Vitals。输出：性能现状评估、瓶颈定位（加载/运行时/渲染）、优化方案（按 ROI 排序）、实施步骤、预期效果、监控建议。防呆要求：如果没有性能数据，基于常见性能问题给出排查清单，引导用户收集数据。」

- 实现逻辑
  - 性能诊断 → 瓶颈定位 → 优化方案 → 实施建议 → 效果验证。

---

4) 无障碍审计 Skill（fe-accessibility-audit）

- 用户提供什么
  - 页面 URL 或代码、技术栈、目标标准（WCAG 2.1 A/AA/AAA）、已知的无障碍问题

- 提示词（可直接用）
  - 「创建 fe-accessibility-audit Skill。对前端页面进行无障碍（a11y）审计，确保符合 WCAG 标准。输出：符合度评分、问题列表（按 WCAG 标准分类）、每个问题的修复方案、修复优先级、验证方法。防呆要求：如果目标标准不明确，默认使用 WCAG 2.1 AA 标准。」

- 实现逻辑
  - 自动化检查 → 手动审计 → 问题分类 → 修复方案 → 验证方法。

---

5) API 集成 Skill（fe-api-integration）

- 用户提供什么
  - API 文档或接口列表、技术栈（框架/HTTP 客户端）、认证方式、缓存需求

- 提示词（可直接用）
  - 「创建 fe-api-integration Skill。设计和实现前端与后端的 API 集成，包括请求封装、错误处理和缓存策略。输出：API 客户端封装代码、TypeScript 类型定义、错误处理策略、缓存方案、请求/响应拦截器、使用示例。防呆要求：如果 API 文档不完整，基于 RESTful 规范推断接口结构，标注[假设]。」

- 实现逻辑
  - API 分析 → 封装设计 → 错误处理 → 缓存策略 → 类型定义。

---

6) 响应式设计 Skill（fe-responsive-design）

- 用户提供什么
  - 设计稿（桌面/平板/手机）、技术栈（CSS 方案）、断点策略、特殊设备需求

- 提示词（可直接用）
  - 「创建 fe-responsive-design Skill。设计和实现响应式页面，确保在各种设备和屏幕尺寸上都有良好的体验。输出：断点定义、各断点布局策略、响应式 CSS/代码、图片适配方案、触摸交互优化、测试清单。防呆要求：如果设计稿只提供了一种尺寸，基于内容优先级推断其他尺寸的布局策略，标注[假设]。」

- 实现逻辑
  - 设计分析 → 断点定义 → 布局策略 → 实现方案 → 测试清单。

---

7) 状态管理 Skill（fe-state-management）

- 用户提供什么
  - 应用类型和复杂度、技术栈、状态类型（用户/UI/表单/数据）、团队经验

- 提示词（可直接用）
  - 「创建 fe-state-management Skill。设计前端状态管理架构，包括全局状态、服务端状态和 URL 状态的划分。输出：状态分类（服务端/全局/URL/表单）、架构设计图、工具选型建议、状态定义和操作、调试和 DevTools 配置。防呆要求：如果应用复杂度不高，推荐简化的状态方案（如 React Context + useReducer），避免过度设计。」

- 实现逻辑
  - 状态分类 → 架构设计 → 工具选型 → 实现方案 → 调试策略。

---

8) 测试策略 Skill（fe-testing-strategy）

- 用户提供什么
  - 技术栈和框架、测试工具（Vitest/Jest/Cypress/Playwright）、覆盖率要求、关键用户流程

- 提示词（可直接用）
  - 「创建 fe-testing-strategy Skill。制定前端测试策略，包括单元测试、集成测试和 E2E 测试的覆盖方案。输出：测试金字塔（单元/集成/E2E 比例）、各层测试重点、工具链配置、关键流程的测试用例、CI/CD 集成方案、覆盖率目标。防呆要求：如果团队测试经验不足，从核心流程开始，逐步扩展测试覆盖。」

- 实现逻辑
  - 风险评估 → 测试金字塔设计 → 工具选型 → 测试用例设计 → CI/CD 集成。

---

以上 8 个 Skill 覆盖了前端工程师的核心工作流。

使用方式：把上面任意一条"提示词"直接发给 Agent（比如 Claude Code），它会帮你生成对应的 Skill 文件。

建议优先从最耗时、最高频的场景开始，做完一个用起来，再做下一个。

如果这份清单里有不符合实际的场景，告诉我，我来替换。
