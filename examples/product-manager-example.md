# 产品经理 Skill 规划输出示例

> 以下是在 Claude Code 中说"我是产品经理，帮我拆 Skill"后，career-skill-planner 输出的完整清单。

---

⚠️ 以下基于「产品经理」的常见职责拆解。如果实际工作有差异，告诉我，我来调整。

---

1) 需求文档编写 Skill（pm-prd-writer）

- 用户提供什么
  - 业务想法的简要描述、业务目标、目标用户群体、已有的相关文档（如有）

- 提示词（可直接用）
  - 「请为我创建一个 pm-prd-writer Skill。目标是把模糊需求转成可评审 PRD。请输出：name/description、触发词、输入模板、工作流（澄清→结构化→补漏→验收）、质量检查清单、失败兜底策略。PRD 必须包含：背景、目标、范围、流程、异常、非功能、埋点、风险、验收标准。风格简洁可执行。防呆要求：收到用户输入后，先检查是否有功能目标和目标用户这两项核心信息；缺少时一次性列出所有缺失项询问用户；其他非核心信息（如埋点方案、灰度策略）基于常识补全并标注[假设]，不阻塞主流程。」

- 实现逻辑
  - 需求澄清问答 → 信息缺口列表 → PRD 主体生成 → 自动补"异常/边界/埋点" → 输出评审版 + 待确认项。

---

2) 产品评审团 Skill（pm-review-board）

- 用户提供什么
  - 待评审的 PRD 文档或原型图、评审标准（可选）、角色视角（可选，默认全角色）

- 提示词（可直接用）
  - 「创建 pm-review-board Skill。模拟多角色评审会，输入 PRD/原型后输出：阻断项、重要项、建议项，及逐条修改建议。必须含"通过条件"和"不可上线条件"，并给出复评清单。」

- 实现逻辑
  - 读取材料 → 多角色并行审阅 → 冲突归并 → 风险分级 → 生成结论与修订建议。

---

3) 用户研究规划 Skill（pm-user-research）

- 用户提供什么
  - 研究目标和要回答的问题、当前产品阶段（概念/开发/上线/迭代）、目标用户群体、可用资源（时间/预算/样本量）

- 提示词（可直接用）
  - 「创建 pm-user-research Skill。设计用户调研方案，包括调研目标、方法选择、用户招募标准、访谈提纲和数据分析方法。输出完整的研究方案文档。防呆要求：如果用户只提供了模糊的研究目标（如"了解用户"），提供常见研究目标选项让用户选择，或给出引导模板帮助用户明确目标。不阻塞流程。」

- 实现逻辑
  - 研究目标澄清 → 方法匹配 → 方案设计 → 执行计划 → 数据分析框架。

---

4) 数据分析 Skill（pm-data-analysis）

- 用户提供什么
  - 业务问题或假设、可用数据源和数据指标、时间范围、期望的分析深度

- 提示词（可直接用）
  - 「创建 pm-data-analysis Skill。将业务问题转化为数据分析任务，设计分析框架，输出数据洞察和可落地的建议。输出包含：分析目标、指标定义、分析框架、SQL/查询、核心发现、行动建议。防呆要求：如果用户未提供数据源信息，基于常见业务场景推荐指标，标注[假设]。」

- 实现逻辑
  - 问题澄清 → 指标定义 → 分析框架设计 → 数据获取方案 → 洞察提炼 → 行动建议。

---

5) 路线图规划 Skill（pm-roadmap-planner）

- 用户提供什么
  - 业务目标和战略方向、时间范围（季度/半年/年）、资源和团队规模、已知约束条件

- 提示词（可直接用）
  - 「创建 pm-roadmap-planner Skill。将业务目标转化为可执行的产品路线图，平衡短期交付和长期愿景。输出：战略对齐、主题划分、各阶段功能列表、里程碑、依赖关系和资源分配。防呆要求：如果用户未提供明确的时间范围，给出常见选项（Q3/Q4/半年/全年）让用户选择。」

- 实现逻辑
  - 目标对齐 → 需求收集 → 优先级排序 → 路线图绘制 → 风险评估 → 沟通方案。

---

6) 需求优先级排序 Skill（pm-requirement-prioritizer）

- 用户提供什么
  - 需求列表（每个需求的简要描述）、可选：每个需求的 RICE 数据、可选：业务目标和约束

- 提示词（可直接用）
  - 「创建 pm-requirement-prioritizer Skill。使用结构化框架（RICE/MoSCoW/Kano）对需求进行优先级排序，输出排序结果和决策依据。输出：排序框架选择及理由、每个需求的评分和计算过程、最终排序结果（P0/P1/P2/P3）、决策日志。防呆要求：如果用户未提供评分数据，提供简化版排序选项（必须做/应该做/可以做/不做）。」

- 实现逻辑
  - 需求录入 → 框架选择 → 打分/评估 → 排序计算 → 决策报告。

---

7) 干系人沟通 Skill（pm-stakeholder-communication）

- 用户提供什么
  - 沟通场景（日常同步/进度汇报/问题升级/方案确认）、干系人角色（高管/研发/运营/客户）、核心信息点

- 提示词（可直接用）
  - 「创建 pm-stakeholder-communication Skill。为不同的干系人场景准备沟通材料，包括同步邮件、汇报 PPT、升级通知等。根据场景生成对应的沟通材料，语言适配受众。防呆要求：如果用户未明确沟通场景和受众，提供常见场景选项。」

- 实现逻辑
  - 场景识别 → 受众分析 → 信息提炼 → 内容组织 → 语言适配 → 输出。

---

8) 迭代复盘 Skill（pm-iteration-review）

- 用户提供什么
  - 迭代周期和时间范围、迭代目标和计划、实际完成情况、团队反馈（可选）

- 提示词（可直接用）
  -「创建 pm-iteration-review Skill。引导团队进行结构化的迭代复盘，识别做得好的地方、待改进的地方和行动项。输出：目标 vs 实际、Keep/Problem 分析、5 Whys 根因分析、行动项（谁/做什么/何时完成）。防呆要求：如果用户未提供迭代数据，基于常见迭代模式给出复盘框架，引导用户填充。」

- 实现逻辑
  - 数据收集 → 结构化复盘 → 根因分析 → 行动项生成 → 跟踪计划。

---

9) A/B 测试设计 Skill（pm-ab-test-design）

- 用户提供什么
  - 要测试的改动或假设、核心指标和成功标准、用户流量和当前指标基线

- 提示词（可直接用）
  - 「创建 pm-ab-test-design Skill。设计完整的 A/B 测试方案，包括假设、实验设计、指标定义、样本量计算和结果解读方法。输出：实验假设、分组设计、指标体系、样本量计算、实验周期、统计显著性要求。防呆要求：如果用户未提供基线指标，基于行业平均值给出参考值并标注[假设]。」

- 实现逻辑
  - 假设建立 → 实验设计 → 指标体系 → 样本量计算 → 执行计划 → 结果解读框架。

---

10) 功能规格说明书 Skill（pm-feature-spec）

- 用户提供什么
  - 功能需求和用户故事、业务规则和逻辑、接口要求、数据存储需求

- 提示词（可直接用）
  - 「创建 pm-feature-spec Skill。为单个功能编写详细的技术实现规格，供研发团队评审和开发。输出：功能概述、用户故事、流程图、接口定义、数据模型、业务规则表、异常处理矩阵。防呆要求：如果技术细节信息不足，基于常见实现模式给出建议方案并标注[假设]。」

- 实现逻辑
  - 需求拆解 → 技术方案设计 → 接口定义 → 数据模型 → 异常处理 → 评审准备。

---

11) 竞品分析 Skill（pm-competitive-analysis）

- 用户提供什么
  - 竞品名称或 URL、分析维度（功能/定价/用户体验/技术等）、我们的产品定位

- 提示词（可直接用）
  - 「创建 pm-competitive-analysis Skill。对竞品进行系统性分析，输出功能对比、策略差异和机会点。输出：竞品概览、功能对比矩阵、定价策略、差异化优势分析、机会点和威胁、行动建议。防呆要求：如果用户未指定竞品，根据产品领域推荐 3-5 个主要竞品。」

- 实现逻辑
  - 竞品识别 → 信息收集 → 多维度对比 → 差异化分析 → 机会/威胁总结。

---

12) 项目回顾 Skill（pm-project-retro）

- 用户提供什么
  - 项目信息和时间范围、项目目标和实际结果、团队反馈、关键事件时间线

- 提示词（可直接用）
  - 「创建 pm-project-retro Skill。在项目结束后进行系统性回顾，总结成功经验、失败教训和改进方向。输出：项目概要、目标达成情况、关键事件时间线、成功因素、失败教训（根因分析）、量化影响、改进行动项。防呆要求：如果项目数据不完整，基于用户提供的信息尽力分析，标注[信息缺失]。」

- 实现逻辑
  - 数据收集 → 时间线还原 → 根因分析 → 经验沉淀 → 改进计划。

---

13) 团队对齐 Skill（pm-team-alignment）

- 用户提供什么
  - 团队目标或 OKR、当前的优先级和分工、已知的分歧或疑问

- 提示词（可直接用）
  - 「创建 pm-team-alignment Skill。确保团队对目标、优先级和分工有统一的理解，输出对齐文档和沟通话术。输出：目标重申、优先级排序、分工和责任矩阵、时间线和里程碑、风险和应对、共识记录模板。防呆要求：如果目标信息模糊，提供常见对齐议题选项。」

- 实现逻辑
  - 目标澄清 → 分工设计 → 沟通材料准备 → 对齐会议引导 → 共识记录。

---

以上 13 个 Skill 覆盖了产品经理的核心工作流。

使用方式：把上面任意一条"提示词"直接发给 Agent（比如 Claude Code），它会帮你生成对应的 Skill 文件。

建议优先从最耗时、最高频的场景开始，做完一个用起来，再做下一个。

如果这份清单里有不符合实际的场景，告诉我，我来替换。
